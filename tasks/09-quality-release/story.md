# Story: Quality And Release

As a developer, I want the project to stay maintainable and release-ready as features are added.

## Scope

- linting
- typechecking
- build
- browser verification
- mobile review
- docs
- cleanup before release

## Acceptance Criteria

- Core commands pass.
- Browser smoke test covers start, play, death, and restart.
- README explains setup.
- Generated artifacts are ignored.
